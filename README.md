### About Me

I'm a Data Engineer focused on taking data that comes from too many places at once — CRMs, campaign systems, event streams, application logs — and turning it into something a team can actually trust.

My work spans the full pipeline: ingesting raw data, cleaning and standardizing it across inconsistent schemas, modeling it for analytics, validating it before it reaches downstream systems, and delivering it through warehouses, dashboards, or streaming outputs.

The common thread across my projects:

**I build data systems that reconcile messy, multi-source data into something reliable.**

My projects span crypto markets, document search, geospatial disaster data, and conversational AI. The domains differ, but the core problem is the same: raw data from multiple sources needs to be unified, validated, and made trustworthy before anyone can build on it.

I'm currently open to full-time Data Engineer and Data/Analytics Engineer roles.

### Portfolio Theme

Each project shows a different part of the same workflow:

- **Ingestion** — moving raw data from APIs, event streams, or files into a pipeline
- **Transformation** — cleaning, joining, and standardizing data with Python, SQL, or PySpark
- **Modeling** — building schemas, incremental tables, and identity-resolved datasets
- **Delivery** — turning processed data into queryable stores, search interfaces, or real-time aggregates
- **Reliability** — validation, schema enforcement, and quality gating before data moves downstream

### Featured Projects

**[CryptoStream](https://github.com/Balerip/CryptoStream)**
Python | PySpark | Kafka | Spark Structured Streaming | Delta Lake | S3

Built a real-time crypto analytics pipeline ingesting Coinbase tick data via Kafka, processing it with Spark Structured Streaming using watermarked windows (1m/5m/15m), and writing OHLC aggregates to Delta Lake on S3 with ACID-compliant, partition-level backfill.

*Why it matters:* Shows real-time streaming architecture, windowed aggregation, and reprocessing design without full-table rewrites.
